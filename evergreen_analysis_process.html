How to perform the evergreen analysis

Data Pull

  • Start the data pull at least 24 hours ahead of time so mParticle has enough time to process everything.
  • There is a 90 day audience retention issue in mParticle. Which means that you will need to ensure the marketing team knows that there will be bleed between the current targetable and holdout audiences
  • Start mParticle export here
    • The audiences that you want are in "Master Media Hold Out Groups":
      1. Control
      2. Braze Control Group
      3. Paid Social Control Group
      4. Paid Display Control Group
    • The "Identities" that you want are:
      1. mParticleID (not necessary but still useful)
      2. Other ID 5 (necessary)

Data Upload

  • Make sure that you have the service account key in your computer, and note that file path down, it'll be important for the file upload process.
  • Download the audiences and move them to the "Data" folder
  • Unzip the audiences in the Data folder
  • Go to the GCP folder here and move each file to archive for records
  • Go to Notebooks/FileUpload.ipynb in JupyterLab, and replace the string that says INSERT_CREDENTIALS_HERE with the path we noted above
  • Run the first cell of the notebook with Shift+Enter & in the second cell, change the file path to point to where you unziped your files. Then, run the second cell
  • Once the notebook has finished running, you should be able to go back to the GCP folder above and see the new files
  • Delete and then create the four tables that are in our script:
    1. res-nbcupea-dev-ds-sandbox-001.dg_sandbox.all_media_holdout
    2. res-nbcupea-dev-ds-sandbox-001.dg_sandbox.paid_social_holdout
    3. res-nbcupea-dev-ds-sandbox-001.dg_sandbox.paid_display_holdout
    4. res-nbcupea-dev-ds-sandbox-001.dg_sandbox.paid_braze_holdout

Holdout & Targetable Table Creations

  • Open BigQuery and run the script found here to get the adobe IDs and signup dates
  • Run the script found here to create the targetable audiences

Analysis SQL

General KPIs

  • Open this file and edit the "campaign_start_period" variable for the beginning of the month you want to look at, then run it
    • The output will include everything you need except save and winback rates which will be in the next step. Download that file, and put it in the Data folder as general_results.csv

Save & Winback Rate

  • Open this file and edit the report_start_date and report_end_date variables for the beginning and end of the month we're looking at, then run it
    • The output will now include the data necessary for the save rate calculations, save it in the Data folder as save_rate_results.csv
    • Make sure the file only has the columns:
      • audiencecohort
      • save_rate_denom
      • saves
      • save_rate
  • Open this file and again edit the report_start_date and the report_end_date variables for the beginning of the month of interest.
    • Similarly save this result under the Data folder as winback_rate_results.csv
    • Make sure the file only has the columns:
      • audiencecohort
      • lapsed_users
      • winback_users
      • winback_rate

Significance Testing

  • Go to Notebooks/SignificanceTestingV2.ipynb in JupyterLab and run all cells. This will give you the p-values for significance testing.
